## Review Types

- `path_pacing_review`: checks whether a module feels like a guided path rather than a topic dump.
- `path_source_strategy_review`: checks whether the source mix is strong enough and not over-reliant on weak metadata.
- `unit_sequence_review`: checks prerequisite order and lesson grouping.
- `unit_checkpoint_review`: checks whether the checkpoint evaluates learning process, not trading outcome.
- `lesson_wording_review`: checks whether lesson copy sounds teachable and original.
- `lesson_source_evidence_review`: checks direct source relevance, license boundary, and authority context.
- `lesson_boundary_review`: blocks advice, signals, broker workflows, auto-trading, performance claims, and real-money guidance.
- `lesson_teaching_quality_review`: checks misconception, practice response, rubric alignment, and need for historical cases.
- `lesson_hand_authoring_review`: checks whether generated lesson structure has been rewritten into hand-authored teaching prose and fact-checked against cited evidence.

## Why This Matters

The product direction is a trading education knowledge browser: teach first, practice second. That requires more than many URLs. It needs reviewed sources, original explanations, safe boundaries, a course sequence, and feedback prompts that improve reasoning.

The queue makes the next work explicit:

1. Expand official and public-document sources.
2. Distill source metadata into concepts and modules.
3. Promote only reviewed, low-risk material into learner-facing lessons.
4. Run review queue checks before treating lessons as usable in the product UI.
5. Keep historical market/news/sentiment cases separate until data rights and source boundaries are clear.

## Boundary

This queue does not approve production readiness. It does not certify course quality. It does not create trading recommendations, live signals, broker workflows, auto-trading, return claims, win-rate claims, or real-money guidance.
